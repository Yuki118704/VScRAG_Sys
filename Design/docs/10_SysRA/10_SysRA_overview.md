# システム要求分析 (System Requirements Analysis - SysRA)

## 目的
システム全体（ハードウェア、ソフトウェア、人を含むすべての構成要素）が「何をすべきか」を定義します。

## 内容
- 利害関係者要求を受けて、システムレベルでの機能要求・非機能要求を明確化
- システムの境界と外部インターフェースの定義
- システムが満たすべき制約条件の整理

## 使用するSysML図
- **⭐ユースケース図 (Use Case Diagram)**: システムの機能的な振る舞いを表現
- **要求図 (Requirement Diagram)**: システム要求の階層と関係性を表現
- **アクティビティ図 (Activity Diagram)**: システムの処理フローを表現
- **シーケンス図 (Sequence Diagram)**: システムの動的な振る舞いを時系列で表現
- **状態機械図 (State Machine Diagram)**: システムの状態遷移を表現
