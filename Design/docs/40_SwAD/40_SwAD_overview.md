# ソフトウェアアーキテクチャ設計 (Software Architectural Design - SwAD)

## 目的
ソフトウェア要求を実現するための、ソフトウェアの内部構造を設計します。

## 内容
- ソフトウェアをモジュール/コンポーネントに分解
- 各モジュールの責任と機能の定義
- モジュール間のインターフェースとデータフロー
- 使用する設計パターンとアーキテクチャスタイル
- クラス図、シーケンス図などの詳細設計

## 使用するSysML図
- **⭐クラス図 (Class Diagram)**: クラス構造、属性、操作、およびクラス間の関係性を表現
- **⭐シーケンス図 (Sequence Diagram)**: オブジェクト間の相互作用を時系列で表現
- **ブロック定義図 (Block Definition Diagram - BDD)**: ソフトウェアのモジュール構成と関係性を表現
- **内部ブロック図 (Internal Block Diagram - IBD)**: モジュール間の接続とデータフローを表現
- **状態機械図 (State Machine Diagram)**: オブジェクトの状態遷移を詳細に表現
- **アクティビティ図 (Activity Diagram)**: アルゴリズムや処理の詳細フローを表現
