# ソフトウェア要求分析 (Software Requirements Analysis - SwRA)

## 目的
システムアーキテクチャ設計で割り当てられたソフトウェア部分について、「何をすべきか」を詳細に定義します。

## 内容
- システム要求から割り当てられたソフトウェアの機能要求
- ソフトウェアの非機能要求（性能、信頼性、保守性など）
- ソフトウェアの外部インターフェース仕様
- データ要求とデータモデル

## 使用するSysML図
- **⭐データフロー図 (Data Flow Diagram - DFD)**: データの流れと変換を表現
- **⭐ブロック定義図 (Block Definition Diagram - BDD)**: ソフトウェアコンポーネントの構造と関係性を表現
- **要求図 (Requirement Diagram)**: ソフトウェア要求の階層と関係性を表現
- **ユースケース図 (Use Case Diagram)**: ソフトウェアの機能的な振る舞いを表現
- **アクティビティ図 (Activity Diagram)**: ソフトウェアの処理フローを表現
- **シーケンス図 (Sequence Diagram)**: ソフトウェアの動的な振る舞いを表現
- **状態機械図 (State Machine Diagram)**: ソフトウェアの状態遷移を表現
